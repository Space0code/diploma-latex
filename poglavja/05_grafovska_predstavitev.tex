\chapter{Grafovska predstavitev podatkov sledilnika pogleda}
\label{sec:grafovska-predstavitev}

To poglavje je eno osrednjih metodoloških poglavij naloge.
Njegov namen je natančno opisati, kako iz časovnega okna meritev sledilnika pogleda nastane graf, ki ga uporabimo kot vhod v grafovsko nevronsko mrežo.

\section{Učni vzorec kot graf}
\label{subsec:ucni-vzorec}

\subsection{Definicija grafa}

Vsak učni vzorec je časovno okno meritev sledilnika pogleda.
Okno pretvorimo v graf
\begin{equation}
    G = (V, E_t^+, E_t^-, E_s, E_f),
\end{equation}
kjer je $V$ množica vozlišč, $E_t^+$ množica časovnih povezav naprej, $E_t^-$ množica časovnih povezav nazaj, $E_s$ množica prostorskih povezav in $E_f$ množica fiksacijskih povezav.

Vozlišča predstavljajo zaporedne meritve sledilnika pogleda znotraj časovnega okna.
Povezave med vozlišči opisujejo različne relacije med meritvami: časovno zaporednost, prostorsko bližino pogleda in pripadnost isti fiksaciji.
En graf predstavlja en učni vzorec.

\subsection{Oznaka grafa}

Vsak graf označimo z oznako pripadajočega časovnega okna.
V nalogi to oznako uporabimo pri ločenih klasifikacijskih nalogah valence in vzburjenosti.
\END_NOTE{Na koncu popravi, če klasifikacijski nalogi nista ločeni, ampak združeni v eno nalogo. Predvidevam, da bosta ločeni, ker mislim, da so tako tudi naredili v MAHNOB članku.}


\section{Vozlišča}
\label{subsec:vozlisca}

\subsection{Definicija vozlišča}

Vsako vozlišče predstavlja eno meritev sledilnika pogleda znotraj časovnega okna:
\begin{equation}
    v_i \leftrightarrow \mathbf{x}_i.
\end{equation}
Če okno vsebuje $n$ veljavnih meritev, potem graf vsebuje $n$ vozlišč.
V osnovni nastavitvi uporabljamo 10-sekundna okna.
Ker so meritve vzorčene s frekvenco 60~Hz, je pričakovano število meritev v polnem oknu
\begin{equation}
    n_{\mathrm{exp}} = 10~s \cdot 60~Hz = 600.
\end{equation}

Grafovska predstavitev naravno podpira spremenljivo število vozlišč.
Grafovska nevronska mreža namreč deluje na množici vozlišč in povezav, končno predstavitev celotnega grafa pa pridobi z agregacijo vozliščnih predstavitev.

\subsection{Značilke vozlišč}
\label{subsec:znacilke-vozlisca}

Vektor značilk vozlišča je
\begin{equation}
    \mathbf{x}_i =
    \left[
        x_i,
        y_i,
        p_{l,i},
        p_{r,i},
        t_i,
        d_i,
        f_{dur,i}
    \right],
\end{equation}
kjer sta $x_i$ in $y_i$ koordinati pogleda, $p_{l,i}$ in $p_{r,i}$ velikosti leve in desne zenice, $t_i$ je normalizirani čas znotraj okna, $d_i$ razdalja med sledilnikom in udeleženčevimi očmi, $f_{dur,i}$ pa trajanje fiksacije, ki ji pripada $i$-ta meritev.

\subsubsection{Časovna značilka}

Časovna značilka $t_i$ modelu poda položaj meritve znotraj okna.

\subsubsection{Koordinate pogleda}

Koordinati $x_i$ in $y_i$ opisujeta položaj pogleda in sta uporabljeni tudi pri konstrukciji prostorskih povezav.

\subsubsection{Značilke zenice}

Značilki $p_{l,i}$ in $p_{r,i}$ opisujeta velikost leve in desne zenice.

\subsubsection{Razdalja do sledilnika in fiksacije}

Značilki $d_i$ in $f_{dur,i}$ opisujeta razdaljo do sledilnika ter trajanje fiksacije pripadajoče meritve.
Manjkajoče vrednosti trajanja fiksacije in meritve, ki ne pripadajo fiksaciji, so kodirane z vrednostjo $f_{dur,i}=0$.

\section{Heterogena grafovska predstavitev}
\label{subsec:heterogeni-graf}

\subsection{Tipi relacij}

Ker različne povezave opisujejo različne odnose med meritvami, graf obravnavamo kot heterogen graf z več tipi povezav.
V trenutni predstavitvi uporabljamo množico relacij
\begin{equation}
    \mathcal{R} =
    \{\texttt{temporal\_forward}, \texttt{temporal\_backward}, \texttt{spatial}, \texttt{fixation}\}.
\end{equation}
Časovne povezave naprej, časovne povezave nazaj, prostorske povezave in fiksacijske povezave so obravnavane kot ločene relacije.

\subsection{Zakaj ločimo relacije}

Ločevanje relacij omogoča, da model za vsak tip relacije uporablja ločeno transformacijo oziroma ločeno funkcijo posredovanja sporočil.
S tem lahko časovno zaporednost, prostorsko bližino in pripadnost isti fiksaciji modeliramo z različnimi parametri, namesto da bi vse povezave obravnavali kot enakovredne. 
S tem tudi dovolimo modelu, da se za različne relacije nauči dajati rezlično težo istim vozliščem. Morda je neko vozlišče zelo pomembno s pogleda časovne zaporednosti, ampak povsem nepomembno z vidika umeščenosti v prostor.

\section{Povezave}

\subsection{Časovne povezave}
\label{subsec:casovne-povezave}

Časovne povezave povezujejo meritve zaporedne v času.
Naj bodo vozlišča v posameznem oknu urejena po času, tako da je $v_i$ $i$-ta meritev v časovnem zaporedju.
Parameter $k_t \in \mathbb{N}$ določa število časovnih sosedov na vsako stran.
V naši privzeti nastavitvi uporabljamo $k_t = 2$. \END_NOTE{preveri končni $k_t$}

\subsubsection{Povezave naprej in nazaj}

Ločimo usmerjene časovne povezave naprej in nazaj:
\begin{align}
    E_t^+ &= \{(v_i, v_j) : 1 \leq j - i \leq k_t\}, \\
    E_t^- &= \{(v_i, v_j) : 1 \leq i - j \leq k_t\}.
\end{align}

Povezave v $E_t^+$ potekajo od prejšnjih proti kasnejšim meritvam, povezave v $E_t^-$ pa od kasnejših proti prejšnjim meritvam.
Ločitev smeri je smiselna, ker ima vrstni red očesnega gibanja pomen.

\subsubsection{Parameter $k_t$}

Parameter $k_t$ določa velikost lokalnega časovnega konteksta in s tem število povezav, ki jih dodamo okrog posamezne meritve. Smiselno lahko zahtevamo $k_t < n$, kjer je $n$ število vozlišč v oknu. Pričakovano število usmerjenih povezav v grafu naprej je $|E_t^+| = k_t \cdot n - \sum_{i=1}^{k_t} (i) = k_t \cdot n - \frac{k_t(k_t+1)}{2} = k_t \cdot (n - \frac{k_t+1}{2})$. Za naš primer \END_NOTE{$k_t=2$ in $n=600$ je to $|E_t^+| = 2 \cdot (600 - \frac{2+1}{2}) = 1197$ }usmerjenih povezav naprej, enako število pa je tudi usmerjenih povezav nazaj. Torej skupno imamo približno $2 k_t \cdot n$ časovnih povezav.

\END_NOTE{
    \begin{itemize}
        \item Vstavi sliko časovnih povezav.
        \item Razmisli o smiselnosti $k_t$ glede na število layerjev GNN .
    \end{itemize}
}


\subsection{Prostorske povezave}
\label{subsec:prostorske-povezave}

Prostorske povezave so neusmerjene povezave, ki povezujejo meritve s podobnim položajem pogleda znotraj istega časovnega okna.
Za vsako vozlišče poiščemo $k_s$ najbližjih sosedov v prostoru koordinat pogleda $(x,y)$.

\subsubsection{Najbližji sosedi v prostoru pogleda}

Prostorsko razdaljo med meritvama $i$ in $j$ definiramo evklidsko:
\begin{equation}
    d_{ij}^{(s)} =
    \sqrt{(x_i - x_j)^2 + (y_i - y_j)^2}.
\end{equation}

Za vsako vozlišče $v_i$ poiščemo $k_s$ najbližjih sosedov v prostoru koordinat pogleda $(x,y)$ z uporabo k-d drevesa. Če je $v_j$ med temi sosedi, dodamo povezavo med vozliščema $v_i$ in $v_j$; ker prostorske povezave obravnavamo kot neusmerjene, povezavo predstavimo z obema usmerjenima povezavama $(v_i,v_j)$ in $(v_j,v_i)$. Na koncu odstranimo podvojene povezave znotraj relacije $E_s$.

\subsubsection{Simetrizacija povezav}

Ker povezave obravnavamo kot neusmerjene, vsaki povezavi $(v_i,v_j)$ dodamo tudi povezavo $(v_j,v_i)$.

\END_NOTE{
    \begin{itemize}
        \item Vstavi sliko prostorskih povezav.
        \item $k_s=2$ v trenutnih poskusih. Preveri končni $k_s$.
    \end{itemize}
}


\subsection{Fiksacijske povezave}
\label{subsec:fiksacijske-povezave}

Fiksacijske povezave povezujejo meritve, ki pripadajo isti zaznani fiksaciji.
Pripadnost fiksaciji določimo z identifikatorjem fiksacije, podanim v \MAHNOB podatkih.

\subsubsection{Redčenje povezav znotraj fiksacije}

Znotraj posamezne fiksacije ne povežemo vseh vozlišč med seboj, saj bi to povzročilo preveliko število fiksacijskih povezav.
Namesto tega uporabimo redkejšo konstrukcijo, ki ohrani informacijo o pripadnosti isti fiksaciji, vendar zmanjša prostorsko in časovno zahtevnost konstrukcije grafa ter učenja na grafu.
To konstrukcijo imenujemo razširjene povezave znotraj fiksacije (angl. \emph{dilated intra-fixation edges}).
Fiksacijo obravnavamo kot zvezno zaporedje vozlišč z istim veljavnim identifikatorjem \texttt{fixation-index}.
Če ima taka fiksacija $F$ vozlišč, jim znotraj fiksacije priredimo lokalne indekse $0,\ldots,F-1$.

Gostoto povezav uravnava parameter $k_f$, ki določa število fiksacijskih povezav, ki jih generira vsako vozlišče.
V trenutni privzeti konfiguraciji uporabljamo $k_f=3$.
Za posamezno fiksacijo najprej izračunamo korak razširitve
\begin{equation}
    s = \max(1, \lfloor F/k_f + 0{,}5 \rfloor).
\end{equation}
Množico kandidatnih odmikov nato definiramo kot
\begin{equation}
    D = \{(1 + q s) \bmod F \mid q = 0,\ldots,k_f-1\}.
\end{equation}
Morebitni odmik $0$ in podvojene odmike odstranimo.
Za vsak odmik $d \in D$ dodamo neusmerjene povezave
\begin{equation}
    \{v_i, v_{(i+d)\bmod F}\},
    \qquad i = 0,\ldots,F-1.
\end{equation}
Pri zapisu grafa za posredovanje sporočil vsako neusmerjeno povezavo predstavimo z dvema usmerjenima povezavama.
Podvojene povezave znotraj relacije $E_f$ odstranimo.

Na primer, pri fiksaciji dolžine $F=30$ in parametru $k_f=3$ dobimo $s=10$.
Vozlišče z lokalnim indeksom $0$ torej povežemo z vozlišči $1, 11$ in $21$.
Odmik $1$ ohrani časovno lokalno komunikacijo znotraj fiksacije, večji odmiki pa omogočijo širjenje informacij tudi med časovno bolj oddaljenimi meritvami iste fiksacije.

Razširjene povezave imajo večji doseg kot zgolj zaporedne fiksacijske povezave, hkrati pa se izognejo kombinatorični eksploziji, ki bi nastala pri polnem kliku.
Pri polnem kliku bi namreč število povezav naraščalo kvadratno z dolžino fiksacije.
Pri analiziranih 10-sekundnih oknih dobimo približno $1.956$ fiksacijskih povezav na graf pri $k_f=2$, $2.908$ pri $k_f=3$, $4.163$ pri $k_f=5$ in $7.273$ pri $k_f=10$.
Pri privzeti izbiri $k_f=3$ so fiksacijske povezave predstavljale približno $45{,}7\,\%$ vseh povezav v grafu (relacije \texttt{temporal}, \texttt{spatial} in \texttt{fixation}).

Izbira $k_f=3$ je zato kompromis med dosegom in gostoto grafa.
Ker vsaka plast GNN omogoča izmenjavo informacij med neposredno povezanimi vozlišči, se doseg znotraj fiksacije povečuje tudi z večanjem števila plasti.
Pri $k_f=3$ vozlišče že v eni plasti prejme informacije iz več delov iste fiksacije, večplastni model pa lahko te informacije dodatno propagira po fiksacijski relaciji.
Večje vrednosti $k_f$ povečajo gostoto grafa, računsko zahtevnost in možnosti za prekomerno glajenje, manjše vrednosti pa graf dodatno razredčijo.
\END_NOTE{
    \begin{itemize}
        \item Vstavi sliko fiksacijskih povezav.
        \item $k_f=3$ v trenutnih poskusih. Preveri končni $k_f$ glede na eksperimente (consider kf=2).
    \end{itemize}
}

\section{Uteži povezav}
\label{subsec:utezi-povezav}

Uteži povezav omogočajo, da model razlikuje med povezavami, ki povezujejo v času in prostoru različno oddaljene meritve.

\subsection{Značilke povezav}

Za vsako povezavo sestavimo vektor značilk, iz katerega model izračuna surovo skalarno oceno povezave:
\begin{equation}
    a_{ij}=f_{\theta}(\mathbf{r}_{ij}),
\end{equation}
kjer je $\mathbf{r}_{ij}$ vektor značilk povezave $e_{ij}$, $f_{\theta}$ pa večslojni perceptron.
Za prostorske in fiksacijske povezave so uporabljene značilke $[t_i, t_j, \Delta t, \Delta x, \Delta y, d_{ij}^{screen}, \Delta d^{eye}_{ij}]$, za časovne povezave pa dodatno še značilka smeri $\rho_{ij}$.
Pri časovni povezavi naprej je $\rho_{ij}=1$, pri časovni povezavi nazaj pa $\rho_{ij}=-1$.
Prostorske in fiksacijske povezave imajo torej 7, časovne povezave pa 8 značilk, iz katerih se izračunajo uteži povezav.

\subsection{Normalizacija uteži}

Surovo skalarno oceno $a_{ij}$ najprej preslikamo v omejeno območje na interval $(-1, 1)$:
\begin{equation}
    \tilde{w}_{ij} = \tanh(a_{ij}).
\end{equation}

Nato uteži normaliziramo po vhodnih povezavah ciljnega vozlišča. 
To pomeni, da za vozlišče $v_j$ upoštevamo povezave, ki imajo $v_j$ za ciljno vozlišče.
Pri neusmerjenih povezavah, tretiramo vse povezave za vhodne in izhodne.
Normalizacijo izvedemo ločeno za vsako relacijo $r$:
\begin{equation}
    w_{ij}^{(r)} =
    \frac{\tilde{w}_{ij}^{(r)}}
    {\sum_{i' \in \mathcal{N}^{-}_r(j)}
    |\tilde{w}_{i'j}^{(r)}| + \epsilon},
\end{equation}
kjer je $\mathcal{N}^{-}_r(j)$ množica izvornih vozlišč, ki so z vozliščem $v_j$ povezana z vhodno povezavo tipa $r$.
Majhna konstanta $\epsilon$ prepreči deljenje z nič.
Takšna normalizacija stabilizira velikost agregiranih sporočil tudi pri vozliščih z večjim številom vhodnih povezav.

Za izračun surovih ocen uporabljamo tri ločene večslojne perceptrone: enega za prostorske povezave, enega za fiksacijske povezave in enega za časovne povezave.
Časovni relaciji naprej in nazaj si delita isti perceptron, ločeni pa sta z dodatno značilko smeri $\rho_{ij}$.
Skalarne uteži povezav se uporabljajo pri konvoluciji \texttt{GCNConv}, ki je privzeta izbira modela.
Pri poskusih z \texttt{GATConv} se te skalarne uteži ne uporabijo, saj \texttt{GATConv} pomembnost povezav modelira z notranjim mehanizmom pozornosti.
