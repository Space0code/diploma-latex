\chapter{Grafovska predstavitev podatkov sledilnika pogleda}
\label{sec:grafovska-predstavitev}

To poglavje je eno osrednjih metodoloških poglavij naloge.
Njegov namen je natančno opisati, kako iz časovnega okna meritev sledilnika pogleda nastane graf, ki ga uporabimo kot vhod v grafovsko nevronsko mrežo.

\section{Učni vzorec kot graf}
\label{subsec:ucni-vzorec}

\subsection{Definicija grafa}

Vsak učni vzorec je časovno okno meritev sledilnika pogleda.
Okno pretvorimo v graf
\begin{equation}
    G = (V, E_t^+, E_t^-, E_s, E_f),
\end{equation}
kjer je $V$ množica vozlišč, $E_t^+$ množica časovnih povezav naprej, $E_t^-$ množica časovnih povezav nazaj, $E_s$ množica prostorskih povezav in $E_f$ množica fiksacijskih povezav.

Vozlišča predstavljajo zaporedne meritve sledilnika pogleda znotraj časovnega okna.
Povezave med vozlišči opisujejo različne relacije med meritvami: časovno zaporednost, prostorsko bližino pogleda in pripadnost isti fiksaciji.
En graf predstavlja en učni vzorec.

\subsection{Oznaka grafa}

Vsak graf označimo z oznako pripadajočega časovnega okna.
V nalogi to oznako uporabimo pri ločenih klasifikacijskih nalogah valence in vzburjenosti.

\section{Vozlišča}
\label{subsec:vozlisca}

\subsection{Definicija vozlišča}

Vsako vozlišče predstavlja eno meritev sledilnika pogleda znotraj časovnega okna:
\begin{equation}
    v_i \leftrightarrow \mathbf{x}_i.
\end{equation}
Če okno vsebuje $n$ veljavnih meritev, potem graf vsebuje $n$ vozlišč.
V osnovni nastavitvi uporabljamo 10-sekundna okna.
Ker so meritve vzorčene s frekvenco 60~Hz, je pričakovano število meritev v polnem oknu
\begin{equation}
    n_{\mathrm{exp}} = 10~s \cdot 60~Hz = 600.
\end{equation}

Grafovska predstavitev naravno podpira spremenljivo število vozlišč.
Grafovska nevronska mreža namreč deluje na množici vozlišč in povezav, končno predstavitev celotnega grafa pa pridobi z agregacijo vozliščnih predstavitev.

\subsection{Značilke vozlišč}
\label{subsec:znacilke-vozlisca}

Vektor značilk vozlišča je
\begin{equation}
    \mathbf{x}_i =
    \left[
        t_i,
        x_i,
        y_i,
        p_{l,i},
        p_{r,i},
        d_i,
        f_{dur,i}
    \right],
\end{equation}
kjer je $t_i$ čas, ki je pretekel od začetka okna, $x_i$ in $y_i$ sta koordinati pogleda, $p_{l,i}$ in $p_{r,i}$ sta velikosti leve in desne zenice, $d_i$ je razdalja med sledilnikom in udeleženčevimi očmi, $f_{dur,i}$ pa trajanje fiksacije, ki ji pripada $i$-ta meritev.

\subsubsection{Časovna značilka}

Časovna značilka $t_i$ modelu poda položaj meritve znotraj okna.

\subsubsection{Koordinate pogleda}

Koordinati $x_i$ in $y_i$ opisujeta položaj pogleda in sta uporabljeni tudi pri konstrukciji prostorskih povezav.

\subsubsection{Značilke zenice}

Značilki $p_{l,i}$ in $p_{r,i}$ opisujeta velikost leve in desne zenice.

\subsubsection{Razdalja do sledilnika in fiksacije}

Značilki $d_i$ in $f_{dur,i}$ opisujeta razdaljo do sledilnika ter trajanje fiksacije pripadajoče meritve.

\section{Povezave}

\subsection{Časovne povezave}
\label{subsec:casovne-povezave}

Časovne povezave povezujejo zaporedne meritve znotraj lokalnega časovnega konteksta.
Naj bodo vozlišča v posameznem oknu urejena po času, tako da je $v_i$ $i$-ta meritev v časovnem zaporedju.
Parameter $k_t \in \mathbb{N}$ določa število časovnih sosedov na vsako stran in ne časovnega radija v sekundah.
V trenutnih poskusih uporabljamo $k_t = 2$.

\subsubsection{Povezave naprej in nazaj}

Ločimo usmerjene časovne povezave naprej in nazaj:
\begin{align}
    E_t^+ &= \{(v_i, v_j) : 1 \leq j - i \leq k_t\}, \\
    E_t^- &= \{(v_i, v_j) : 1 \leq i - j \leq k_t\}.
\end{align}

Povezave v $E_t^+$ potekajo od prejšnjih proti kasnejšim meritvam, povezave v $E_t^-$ pa od kasnejših proti prejšnjim meritvam.
Ločitev smeri je smiselna, ker ima vrstni red očesnega gibanja pomen.

\subsubsection{Parameter $k_t$}

Parameter $k_t$ določa velikost lokalnega časovnega konteksta in s tem število povezav, ki jih dodamo okrog posamezne meritve.

\subsection{Prostorske povezave}
\label{subsec:prostorske-povezave}

Prostorske povezave so neusmerjene povezave, ki povezujejo meritve s podobnim položajem pogleda znotraj istega časovnega okna.
Za vsako vozlišče poiščemo $k_s$ najbližjih sosedov v prostoru koordinat pogleda $(x,y)$.

\subsubsection{Najbližji sosedi v prostoru pogleda}

Prostorsko razdaljo med meritvama $i$ in $j$ definiramo kot
\begin{equation}
    d_{ij}^{(s)} =
    \sqrt{(x_i - x_j)^2 + (y_i - y_j)^2}.
\end{equation}
Povezava $(v_i, v_j)$ je dodana, če je $v_j$ med $k_s$ najbližjimi prostorskimi sosedi vozlišča $v_i$.

\subsubsection{Simetrizacija povezav}

Ker povezave obravnavamo kot neusmerjene, vsaki povezavi $(v_i,v_j)$ dodamo tudi povezavo $(v_j,v_i)$.

\subsection{Fiksacijske povezave}
\label{subsec:fiksacijske-povezave}

Fiksacijske povezave povezujejo meritve, ki pripadajo isti zaznani fiksaciji.
Pripadnost fiksaciji določimo z identifikatorjem fiksacije.

\subsubsection{Redčenje povezav znotraj fiksacije}

Znotraj posamezne fiksacije ne povežemo vseh vozlišč med seboj, saj bi to povzročilo veliko število povezav pri daljših fiksacijah.
Namesto tega uporabimo redkejšo konstrukcijo, ki ohrani informacijo o pripadnosti isti fiksaciji, vendar zmanjša prostorsko in časovno zahtevnost konstrukcije grafa.

\section{Uteži povezav}
\label{subsec:utezi-povezav}

Uteži povezav omogočajo, da model razlikuje med bližnjimi in oddaljenimi povezavami.

\subsection{Značilke povezav}

V trenutni smeri razvoja uporabimo naučene uteži povezav, izračunane iz značilk relacije:
\begin{equation}
    w_{ij}=f_{\theta}(\mathbf{r}_{ij}),
\end{equation}
kjer je $\mathbf{r}_{ij}$ vektor značilk povezave, $f_{\theta}$ pa večslojni perceptron.
Za prostorske povezave so priporočene značilke $[t_i, t_j, \Delta t, \Delta x, \Delta y, d_{ij}]$, za časovne povezave pa dodatno še značilka smeri.

\section{Heterogena grafovska predstavitev}
\label{subsec:heterogeni-graf}

\subsection{Tipi relacij}

Ker različne povezave opisujejo različne odnose, graf obravnavamo kot heterogen graf z več tipi povezav.
Prostorske povezave, časovne povezave naprej in časovne povezave nazaj imajo ločene relacije.

\subsection{Zakaj ločimo relacije}

To omogoča, da model za vsak tip odnosa uporablja ločeno transformacijo oziroma ločeno funkcijo posredovanja sporočil.
